ATTENDANCE ────→ NOTIFICATIONS (via trigger)


┌─────────────────────────────────────────────────────────────────────────────┐
│                         KEY CONSTRAINTS                                      │
└─────────────────────────────────────────────────────────────────────────────┘

1. CASCADE DELETE: Deleting a user automatically deletes:
   - Associated student/teacher record
   - All attendance records (if student)
   - All notifications

2. UNIQUE CONSTRAINTS:
   - username (must be unique)
   - email (must be unique)
   - roll_number (must be unique)
   - employee_id (must be unique)
   - (student_id, date) in attendance table (one record per day)

3. FOREIGN KEY CONSTRAINTS:
   - All student records must have valid user_id
   - All teacher records must have valid user_id
   - All attendance records must have valid student_id and teacher_id
   - All notifications must have valid user_id

┌─────────────────────────────────────────────────────────────────────────────┐
│                         DATA FLOW                                            │
└─────────────────────────────────────────────────────────────────────────────┘

User Registration Flow:
1. Insert into USERS table
2. Insert into STUDENTS or TEACHERS table (based on user_type)

Attendance Marking Flow:
1. Teacher marks attendance → Insert/Update ATTENDANCE table
2. Trigger fires → Auto-create NOTIFICATION
3. Student logs in → View NOTIFICATION

Attendance Query Flow:
1. Student/Teacher queries → Use VIEWS for optimized results
2. Complex reports → Use STORED PROCEDURES
3. Real-time stats → Direct table queries with indexes
