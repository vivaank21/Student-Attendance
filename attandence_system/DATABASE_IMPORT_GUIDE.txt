DATABASE IMPORT GUIDE
====================

This guide will help you import the attendance_system.sql database file.

METHOD 1: Using MySQL Command Line (Recommended)
=================================================

Step 1: Open Terminal/Command Prompt

Step 2: Navigate to the directory containing attendance_system.sql
    cd path/to/your/files

Step 3: Import the database
    mysql -u root -p < attendance_system.sql

    (You'll be prompted to enter your MySQL password)

Step 4: Verify the import
    mysql -u root -p
    USE attendance_system;
    SHOW TABLES;
    SELECT COUNT(*) FROM users;
    EXIT;


METHOD 2: Using MySQL Workbench (GUI)
======================================

Step 1: Open MySQL Workbench

Step 2: Connect to your MySQL server

Step 3: Click on "Server" menu → "Data Import"

Step 4: Select "Import from Self-Contained File"

Step 5: Browse and select "attendance_system.sql"

Step 6: Under "Default Target Schema" → Select "New..." and name it "attendance_system"

Step 7: Click "Start Import"

Step 8: Verify by checking tables in the Schemas panel

===========================================

Step 1: Open phpMyAdmin in your browser

Step 2: Click on "Import" tab

Step 3: Click "Choose File" and select "attendance_system.sql"

Step 4: Make sure format is set to "SQL"

Step 5: Click "Go" at the bottom

Step 6: Wait for import to complete

TROUBLESHOOTING
===============

Error: "Access denied"
Solution: Make sure you're using the correct MySQL username and password

Error: "Database already exists"
Solution: The SQL file drops and recreates the database, so this shouldn't happen
         If it does, manually drop it first: DROP DATABASE attendance_system;

Error: "Table already exists"
Solution: Run: DROP DATABASE attendance_system;
         Then import again

Error: "Unknown database"
Solution: Make sure MySQL server is running

Error: Import takes too long
Solution: This is normal for large datasets. Wait for completion.


BACKUP YOUR DATABASE
====================

To restore from backup:
